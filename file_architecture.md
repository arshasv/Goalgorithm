# GOALGORITHM - File Architecture Documentation

This document explains the purpose and responsibilities of the critical files within the GOALGORITHM project. The system is split between a Python/FastAPI backend and a React/Vite frontend.

---

## 1. Backend File Architecture (`backend/app/`)

The backend follows a layered architecture: **Routes → Services → Repositories → Models**.

### 1.1 Entry Point & Core Configuration
- **`main.py`**: The entry point of the FastAPI application. It initializes the FastAPI instance, registers CORS middleware to allow frontend communication, binds the exception handlers, and mounts all the API routers from the `api/` directory.
- **`config.py`**: Manages environment variables and application settings using Pydantic's `BaseSettings`. Handles secret keys, database URIs, and external API configurations.
- **`database/connection.py`** & **`database/session.py`**: Configures the SQLAlchemy `engine` and `sessionmaker`. Sets up the connection pool to the PostgreSQL database.
- **`database/base.py`**: Defines the `Base` declarative class that all SQLAlchemy models inherit from.

### 1.2 Database Models (`models/`)
Defines the SQL tables using SQLAlchemy ORM.
- **`user.py`**: Defines `UserModel`. Stores Organizer and Team Leader accounts, handling roles and hashed passwords.
- **`team.py`**: Defines `TeamModel`. Stores team names, codes, and maps teams to their respective Team Leaders.
- **`match.py`**: Defines `MatchModel`. Stores match metadata, freeze deadlines, kick-off times, and sync statuses.
- **`prediction.py`**: Defines `PredictionModel` and `PlayerPredictionModel`. Stores the JSON payloads submitted by Team Leaders (predicted winner, scoreline, goalscorers).
- **`score.py`**: Defines `ScoreModel` and `CumulativePhaseScoreModel`. Stores the mathematically evaluated base scores and multipliers generated by the scoring engine.
- **`actual_result.py`**: Defines `ActualResultModel`. Stores the real-world outcome of matches fetched from the external Football API.
- **`evaluation.py`**: Defines `TechnicalEvaluationModel` and `PresentationEvaluationModel` used by judges to grade teams.
- **`leaderboard.py`**: Defines `LeaderboardModel`. Aggregates all phase scores into a final ranking out of 100 points.

### 1.3 Data Access Layer (`repositories/`)
Abstracts raw SQL and database queries away from the business logic.
- **`base_repository.py`**: A generic generic base class containing standard CRUD operations (`create`, `get`, `update`, `delete`).
- **`match_repository.py`**: Handles custom queries for fetching matches by status, external API ID, or sync requirement.
- **`prediction_repository.py`**: Queries to fetch predictions associated with a specific match or team.
- **`score_repository.py`**: Queries the latest leaderboard and score calculations.

### 1.4 Business Logic (`services/`)
Where the core operations, logic checks, and API orchestrations live.
- **`auth_service.py`**: Handles hashing passwords with Passlib, verifying credentials, and generating JWT tokens using the `jose` library.
- **`match_service.py`**: Orchestrates match creation, deletion, and triggering `FootballAPIService` to sync external matches. Contains cascading deletion logic to clean up orphan predictions and scores if a match is removed.
- **`prediction_service.py`**: Validates prediction submissions against the match's `freeze_deadline`. Inserts payload to DB via `PredictionRepository`.
- **`scoring_service.py`**: The orchestrator that fetches predictions and actual results, and passes them to the pure `scoring_engine`. Saves the output to `ScoreModel`.
- **`report_service.py`**: Aggregates scoring data across multiple tables to generate the "Score Journey" transformation dictionary used by the Organizer Reports dashboard.
- **`football_api_service.py`**: Connects to the external API-Sports / Sportmonks provider to download fixtures and live results.

### 1.5 Scoring Engine (`scoring_engine/`)
Pure, deterministic mathematical calculation layer. No DB or HTTP calls happen here.
- **`base_score/winner_score.py`**: Calculates points awarded if the predicted winner matches the actual winner.
- **`base_score/scoreline_score.py`**: Awards high points for exactly predicting the final goals of both teams.
- **`base_score/probability_score.py`**: Calculates mathematical accuracy (e.g., Brier score) based on submitted win/draw/loss percentages.
- **`multiplier/ranking_engine.py`**: Takes the raw base scores of all teams for a match, ranks them, and applies a Grade Multiplier (A=3x, B=2x, etc.).
- **`normalization/phase1_normalizer.py`**: Scales the total cumulative Phase 1 (prediction) points linearly down to the maximum allowed Phase 1 contribution (e.g., 20 points).

### 1.6 API Controllers (`api/`)
FastAPI routers that define HTTP endpoints, inject dependencies (DB sessions, current user), and call services.
- **`deps.py`**: Contains dependency injection functions like `get_db` and authentication guards like `get_current_organizer` and `get_current_team_leader`.
- **`auth_routes.py`**: `POST /login` endpoint.
- **`match_routes.py`**: `GET`, `POST`, `PUT`, `DELETE` endpoints for matches. Includes the `/sync` trigger.
- **`prediction_routes.py`**: Endpoints for Team Leaders to submit JSON predictions.
- **`report_routes.py`**: Endpoints to generate the UI visualization breakdown of the scoring journey.

## 2. Frontend File Architecture (`react-frontend/src/`)

The frontend is a React SPA built with Vite.

### 2.1 Entry Point & Core Configuration
- **`main.jsx`**: The React DOM mounting point. Wraps the application in the `BrowserRouter` and `AuthProvider`.
- **`App.jsx`**: The root component. Defines all application routes using `react-router-dom`. Applies the `<Layout>` wrapper and enforces `<ProtectedRoute>` wrappers around authenticated pages.
- **`index.css`**: Contains global CSS variables, custom themes (Dark Navy, Muted Gold), and standard styling utility classes.

### 2.2 Global State & Context (`contexts/`)
- **`AuthContext.jsx`**: Manages the user's session state. Reads the JWT token from `localStorage`, decodes the user's role (Organizer vs Team Leader), and exposes `login()`, `logout()`, and `user` state globally.

### 2.3 API Integration (`api/`)
Axios wrapper classes that handle network requests to the FastAPI backend.
- **`axios.js`**: Configures the base Axios instance. Attaches the `Authorization` bearer token to outgoing requests and catches 401 Unauthorized errors to trigger automatic logouts.
- **`authService.js`**: Calls `/api/v1/auth/login`.
- **`matchService.js`**: Fetches the list of matches, triggers syncs, and handles match deletion.
- **`predictionService.js`**: Submits match prediction payloads to the backend.
- **`reportService.js`**: Fetches the aggregated scoring journey reports.

### 2.4 Views / Pages (`pages/`)
The main container components rendered by the Router.
- **`auth/LoginView.jsx`**: The login screen form.
- **`dashboard/OrganizerDashboard.jsx`**: High-level system overview, quick metrics, and configuration links for the Organizer.
- **`dashboard/TeamLeaderDashboard.jsx`**: Focused view showing the team's upcoming matches and active model status.
- **`matches/MatchesView.jsx`**: Renders the table/grid of fixtures. Handles different UI states based on role (e.g., Organizers see total prediction counts, Team Leaders only see their own submission status).
- **`predictions/PredictionsView.jsx`**: Shows the history of submitted predictions. Includes logic to safely hide raw UUIDs if a dependent match was deleted (shows "Deleted Match").
- **`reports/ReportsView.jsx`**: Connects to the ReportService. Visualizes the Team Score Journey using downward arrows mapping Original Score to Leaderboard Contribution.
- **`analytics/AnalyticsView.jsx`**: Heavy utilization of the `recharts` library to render responsive Bar and Line charts showing Model Performance and Judge Analytics. Enforces strict CSS height limits and branding colors.
- **`leaderboard/LeaderboardView.jsx`**: Displays the final computed ranking of all teams out of 100 points.

### 2.5 Reusable Components (`components/`)
- **`layout/Sidebar.jsx`**: The left navigation pane. Dynamically renders links based on `AuthContext` roles.
- **`layout/Navbar.jsx`**: The top bar, housing the user profile dropdown and logout button.
- **`predictions/SubmitPredictionModal.jsx`**: A complex form component enforcing client-side validation (e.g., probabilities must equal 100) before submitting to the backend.
- **`matches/MatchDetailModal.jsx`**: Displays read-only match information.
