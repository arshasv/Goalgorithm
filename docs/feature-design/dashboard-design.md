# Dashboard UX Design

## Overview
The Dashboard serves as the central hub for users. The experience is strictly bifurcated based on the user's role to ensure relevance and security.

## Organizer Dashboard
**Purpose**: High-level oversight of the competition.
- **Competition Overview**: Top-level widgets displaying total teams, scheduled matches, and overall evaluation progress.
- **Quick Metrics**: Key performance indicators such as the current leaderboard leader, highest scores, and recent model submissions.
- **Navigation**: Sidebar provides full access to Team Management, Match Management, Scoring Configuration, Evaluations, Analytics, and Reports.
- **System Status**: Indicators showing whether the external API synchronization is active or pending.

## Team Dashboard
**Purpose**: Focused execution and tracking for a specific team.
- **Matches Available**: A streamlined view of upcoming fixtures where predictions or models can be applied.
- **Prediction Status**: Clear indicators showing whether predictions for upcoming matches have been submitted or are pending.
- **Model Status**: An overview of the team's current active model, including upload date and evaluation status (e.g., Training, Ready, Failed).
